# Product Catalog Application

Это консольное приложение для управления каталогами товаров. 
Оно позволяет просматривать, добавлять, редактировать и удалять каталоги и продукты, с поддержкой аутентификации пользователей, аудита действий и фильтрации продуктов.
Бэкенд использует PostgreSQL как базу данных, Liquibase для миграций, и реализовано на Java.

## Функциональность

- **Пользователи**: Регистрация, логин/логаут. Роли: User (просмотр) и Admin (полное управление).
- **Каталоги**: Создание, удаление, просмотр.
- **Продукты**: Добавление, обновление, удаление, фильтрация (по имени, цене, бренду, категории).
- **Аудит**: Логирование действий в БД.
- **Метрики**: Отображение времени выполнения операций и количества каталогов.
- **UI**: Консольный интерфейс с меню.

## Требования

- **Java**: 17+.
- **Maven**: 3.8+ для сборки и тестирования.
- **Docker**: Для запуска PostgreSQL в контейнере.
- **PostgreSQL**: Версия 16+.
- **Дополнительно**: Для тестов требуется Docker (Testcontainers использует контейнеры для изоляции).

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/LittleBlBb/Y_LAB_HOMEWORK.git
cd Y_LAB_HOMEWORK
```

### 2. Настройка конфигурации

Файл `src/main/resources/application.properties` содержит настройки БД. По умолчанию:

```
liquibase.changeLog=db/changelog/changelog.xml
liquibase.serviceSchema=liquibase

db.host=localhost
db.username=catalog
db.password=password
db.name=postgres
db.port=15432
db.schema=app
```

### 3. Запуск базы данных в Docker

Проект использует PostgreSQL в Docker. Файл `docker-compose.yml` настроен для запуска контейнера.

- Запустите Docker Compose:

```bash
docker-compose up -d
```

Это запустит PostgreSQL на порту 15432 с пользователем `catalog`, паролем `password` и БД `postgres`.

- Проверьте статус:

```bash
docker ps
```

Должен отображаться контейнер `postgres`.

- Остановка:

```bash
docker-compose down
```

### 4. Применение миграций Liquibase

Миграции создают схему `app` для моделей и `liquibase` для служебных таблиц, также создаются таблицы, последовательности и вставляют начальные данные (админ-аккаунт, каталоги, продукты).

- Соберите проект Maven (если не собрали):

```bash
mvn clean install
```

- Запустите класс `Migrator` (находится в `ProductCatalog.DB.Migrator`):
  - Через IDE: Запустите `Migrator.main()` с настройками JVM (если нужно).

- Ожидаемый вывод: "Migration is completed successfully".

### 5. Сборка и запуск приложения

- Соберите проект:

```bash
mvn clean package
```

- Запустите главное приложение (`Main` в `ProductCatalog.Main`):
  - Через IDE: Запустите `Main.main()`.

Приложение запустится в консоли. Используйте меню для навигации.

#### Использование UI

- Главное меню: Выберите опции цифрами (1 - Каталоги, 2 - Вход, 3 - Регистрация, 0 - Выход).
- По умолчанию есть админ: логин `admin`, пароль `admin`.
- Для админа доступны: добавление/удаление/редактирование, журнал аудита (опция 5).
- Фильтры: В меню продуктов нажмите "F" для фильтрации.

### 6. Запуск тестов

Тесты используют JUnit и Testcontainers (автоматически запускают PostgreSQL в контейнере для изоляции).

- Запустите все тесты:

```bash
mvn test
```

- Для конкретного теста (через IDE): Запустите классы в `src/test/java` (например, `AuditRepositoryTest`).
- Требования: Docker должен быть запущен, так как Testcontainers создает контейнеры на лету.

## Структура проекта

- `src/main/java/ProductCatalog/`: Модели, Репозитории, Сервисы, UI, DB (конфиг, мигратор).
- `src/main/resources/`: `application.properties` (конфиг), Liquibase changelogs.
- `src/test/java/`: Unit-тесты для репозиториев.
- `docker-compose.yml`: Для PostgreSQL.
